## 🎬 시연 시나리오 (발표 흐름)

```
┌─────────────────────────────────────────────────────────┐
│            발표 시연 Step-by-Step 가이드                  │
└─────────────────────────────────────────────────────────┘
```

### Step 1: 서버 초기 설정 — `/방생성`

```
/방생성
```

**무슨 일이 일어나나?**
- 📚 학급 카테고리 아래 1-1 ~ 3-4 텍스트 채널 12개 자동 생성
- 각 반 역할(1-1, 1-2, ... 3-4) 자동 생성 및 채널 권한 설정
- 👩‍🏫 교직원 카테고리 + 선생님 전용 채널 생성
- 🔒 관리 카테고리 + 콘솔·로그 채널 생성 (로그는 봇만 쓰기 가능)
- 팀원모집 채널 생성 (전체 읽기 가능)

**시연 포인트**: 명령어 하나로 복잡한 서버 구조가 자동으로 만들어지는 것을 보여줌

---

### Step 2: 가입 및 승인 시스템 — `/가입`

```
/가입 이름:홍길동 생년월일:2009-03-01 반:2-3
```

**무슨 일이 일어나나?**
1. 가입 요청이 DB의 `registrations` 테이블에 저장됨
2. 해당 반 채널에 **승인/거절 버튼이 달린 임베드** 자동 전송
3. `2-3-관리자` 역할 보유자가 버튼 클릭으로 승인
4. 승인 시 사용자에게 `2-3` 역할 자동 부여 + 로그 채널에 기록

**시연 포인트**: 직접 버튼 클릭으로 승인 과정 시연

---

### Step 3: 과제 등록 및 자동 알림 — `/과제추가`

```
/과제추가 제목:알고리즘 수행평가 마감:2026-06-20 23:59 과목:정보처리 제출:클래스룸
```

**무슨 일이 일어나나?**
1. 과제가 `assignments` 테이블에 저장
2. `notification_rules`에 따라 알림 스케줄 자동 생성

```
알림 예약 스케줄:
  📅 2026-06-13 23:59  ← 7일 전 알림
  📅 2026-06-17 23:59  ← 3일 전 알림
  📅 2026-06-19 23:59  ← 1일 전 알림
  📅 2026-06-20 22:59  ← 1시간 전 알림
```

3. 지정된 시각에 반 채널로 자동 알림 발송

**시연 포인트**: 과제 등록 → `/과제목록`으로 남은 시간 확인

---

### Step 4: 시간표 조회 — `/시간표`

```
/시간표 반:2-3
```

**무슨 일이 일어나나?**
- NEIS 교육정보 개방포털 API 호출
- 학급별 주간 시간표를 파싱하여 Discord 임베드로 출력
- 요일별 교과목 표시

**시연 포인트**: 실제 학교 데이터와 연동된 시간표 실시간 조회

---

### Step 5: 급식 조회 — `/급식`

```
/급식
/급식 날짜:2026-06-10
```

**무슨 일이 일어나나?**
- NEIS API에서 해당 날짜 급식 데이터 조회
- 아침/점심/저녁 메뉴 + 칼로리 정보를 임베드로 표시
- 날짜 미입력 시 오늘 급식 자동 표시

**시연 포인트**: 학생들이 가장 자주 쓸 실용 기능 시연

---

### Step 6: 실시간 투표 — `/투표`

```
/투표 제목:"체육대회 종목 선택" 선택지:"이어달리기,줄다리기,피구,릴레이"
```

**무슨 일이 일어나나?**
1. 투표 임베드가 채널에 게시됨
2. 각 선택지가 버튼으로 표시됨
3. 클릭할 때마다 실시간 집계
4. `/투표종료` 실행 시 최종 결과 및 통계 표시

**시연 포인트**: 실시간 버튼 투표 시연 + 결과 확인

---

### Step 7: 팀 프로젝트 — `/팀원모집`

```
/팀원모집 팀명:"AI 프로젝트팀" 설명:"머신러닝 과제 팀원 구합니다" 최대인원:4
```

**무슨 일이 일어나나?**
1. 팀원모집 채널에 모집 임베드 게시 (참가 버튼 포함)
2. 팀 프로젝트용 채팅 채널(`AI프로젝트팀-채팅`) + 음성 채널 자동 생성
3. 참가 버튼 클릭 시 팀장에게 승인 요청 알림
4. 승인 시 팀원에게 팀 채널 접근 권한 자동 부여
5. `/팀완료` 후 익명 동료 평가(1~5점) 진행

**시연 포인트**: 버튼 클릭 참가 → 팀 채널 자동 생성 프로세스

---

### Step 8: 서버 현황 — `/서버현황`

```
/서버현황
```

**무슨 일이 일어나나?**
- 전체 멤버 수 / 온라인 멤버 수
- 채널 수 (텍스트 / 음성)
- 역할 수
- 학급별 학생 수 통계
- 팀 프로젝트 / 동아리 현황
- 모두 Discord 임베드 형식으로 출력

**시연 포인트**: 학교 서버 전체 현황을 한눈에 확인

## 🔐 보안 설계

| 항목 | 구현 방법 |
|------|-----------|
| **채널 이름 위장 방지** | 채널 이름 대신 Discord Channel ID(숫자)로 인증 → 이름을 바꿔도 권한 우회 불가 |
| **역할 기반 접근 제어 (RBAC)** | `선생님` 역할 / `<반>-관리자` 역할 / Discord 관리자 권한 3단계 분리 |
| **승인 권한 체크** | 팀장(팀 참가 승인) / 선생님(관리 명령어) / 반관리자(가입 승인) 역할별 권한 분리 |
| **관리자 명령어 채널 제한** | 위험 관리자 명령어는 `콘솔` 채널에서만 실행 가능 |
| **위험 명령어 확인 키워드** | 일괄 추방·채널 전체 삭제 등 위험 작업은 확인 키워드 직접 입력 필요 |
| **로그 채널 쓰기 제한** | 로그 채널은 봇만 메시지 전송 가능, 선생님은 읽기만 가능 |
| **비밀값 분리** | 봇 토큰·DB URL·AI 토큰 등 모든 비밀값은 `.env`에만 보관, `config.json`에 절대 포함 금지 |
| **모더레이션 감사 로그** | 모든 관리자 작업은 `discord_moderation_logs` 테이블에 자동 기록 |
| **봇 역할 위치 검증** | 봇 역할이 수정 대상 역할보다 낮으면 명령어 자동 차단 |
| **명령어 실행 로그** | 전체 명령어 실행 내역이 `command_logs` 테이블 + 로그 채널에 동시 기록 |
